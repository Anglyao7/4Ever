# AI 工作流助手使用指南

## 概述

AI 工作流助手是灵感画布里的辅助编排功能。它读取当前节点库，把用户的自然语言目标转换成画布节点和连线。

当前实现位于 `frontend/src/AIWorkflowAssistant.tsx`，入口由 `frontend/src/InspirationCanvasPanel.tsx` 渲染。

## 使用前提

- 中枢里需要有可用模型配置。
- 登录且模型配置由后端托管时，助手会使用 `profileId` 调用后端聊天接口。
- 未登录或本地配置场景下，助手继续使用本地模型配置调用聊天接口。

## 使用方式

1. 打开灵感画布。
2. 点击工具栏的“显示 AI 助手”。
3. 输入目标，例如：
   - “读取和某个联系人的会话，让 AI 整理回复，然后发送消息”
   - “生成一张图片，然后作为附件发送给联系人”
   - “测试中枢模型接口，再获取模型列表”
   - “读取地图记忆和 Token 统计，整理后送入秩序”
4. 助手返回包含 JSON 的回复后，前端会解析节点并添加到当前画布。

## 工作流生成格式

助手期望模型返回如下结构：

```json
{
  "nodes": [
    {
      "type": "trigger",
      "label": "手动触发",
      "config": {}
    },
    {
      "type": "ai-chat",
      "label": "AI 对话",
      "config": {
        "prompt": "整理上下文"
      }
    }
  ],
  "connections": [
    { "from": 0, "output": "开始", "to": 1, "input": "输入" }
  ],
  "explanation": "先手动触发，再用 AI 整理上下文。"
}
```

### 解析规则

- `nodes[*].type` 必须存在于 `frontend/src/lib/node-templates.ts`。
- 未知节点类型会被忽略，不会创建到画布上。
- 节点配置会合并模板默认配置和 AI 返回的 `config`。
- `connections` 支持按节点索引声明连线。
- 如果模型没有返回有效 `connections`，前端会按节点顺序自动生成相邻连线。

## 当前节点范围

助手提示词会从 `nodeTemplates` 动态生成节点知识库。当前可用范围包括：

- 手动触发。
- 笔记新增、保存、删除、导出、检索。
- 读取会话、发送消息、发送附件、AI 对话。
- 中枢模型列表读取和接口测试。
- 图片生成。
- 地图记忆和 Token 统计读取。
- 送入秩序。

不要在提示词或文档示例里假设不存在的定时触发、HTTP 请求、条件判断、循环、延迟、Webhook、云盘或表格节点。

## 已实现

- 多轮对话上下文会在当前面板会话中传给模型。
- 结构化 JSON 解析。
- 自动创建画布节点。
- 支持 AI 返回显式 `connections`。
- 缺少连接时自动按节点顺序补线。
- 生成结果可继续由画布的拖拽、删除、手动连线和整理布局能力调整。

## 当前限制

- 对话历史只存在于当前 React 状态，刷新页面后会清空。
- 助手只向当前画布追加新节点，不会直接修改或删除已有节点。
- 助手不会验证节点业务参数是否能在真实后端执行，只保证节点类型来自当前节点库。
- 节点配置仍主要依赖默认值，复杂参数需要后续配置面板支持。
